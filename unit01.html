What is Angular?

Angular is a powerful, open-source web application framework developed by Google. It is designed to build dynamic, single-page applications (SPAs) with a rich user experience. Angular simplifies the development process by providing a robust structure for managing data binding, dependency injection, and modular development.

Key features of Angular include its two-way data binding, component-based architecture, and comprehensive set of tools for testing, routing, and form handling. Angular’s declarative approach to building user interfaces and its extensive ecosystem make it a popular choice for developers creating complex web applications.

What Can You Build with Angular?

Angular enables developers to build a wide range of applications, from simple single-page apps to complex enterprise solutions. Here are some examples of what you can create with Angular:

  • E-commerce platforms with interactive product catalogs and secure checkout processes.
  • Content management systems (CMS) for managing and displaying content.
  • Customer relationship management (CRM) systems with advanced data visualization.
  • Real-time data dashboards for monitoring and analytics.
  • Social media applications with dynamic content feeds and user interactions.

Angular’s rich feature set and scalability make it an excellent choice for building both large-scale and small-scale applications.

Components

Components are the fundamental building blocks of Angular applications. Each component is an encapsulated unit of the user interface and consists of three main parts:

  • Template: Defines the view of the component using HTML.
  • Class: Contains the logic and data for the component written in TypeScript.
  • Styles: Provides styling to the component’s view using CSS.

Angular uses a declarative approach to building UIs by combining these three parts into a single, cohesive unit. This modular structure promotes reusability and maintainability.

Creating a Component

To create a new component, use the Angular CLI command ng generate component. This command generates a component with a TypeScript class, HTML template, and CSS file. 


ng generate component my-component

This will create a new component called MyComponent in the src/app/my-component directory. The component class contains the logic and data, while the HTML template defines the structure, and the CSS file provides the styling.

Example: Simple Component

Here’s a basic example of a component in Angular: 


import { Component } from '@angular/core';

@Component({
  selector: 'app-hello-world',
  template: '<h1>Hello, {{ name }}!</h1>',
  styles: ['h1 { color: blue; }']
})
export class HelloWorldComponent {
  name: string = 'Angular';
}

In this example, the HelloWorldComponent displays a simple greeting message. The name property is bound to the template, showing "Hello, Angular!" with blue text.

Directives

Directives are special markers in Angular templates that extend HTML functionality. They can manipulate the DOM or enhance the behavior of elements. Angular has several types of directives:

  • Structural Directives: Change the structure of the DOM by adding or removing elements, such as *ngIf* and *ngFor*.
  • Attribute Directives: Change the appearance or behavior of elements, such as the *ngClass* directive.
  • Custom Directives: User-defined directives that can be created to encapsulate reusable behavior.

Using Built-in Directives

Angular provides several built-in directives. For example: 


<div *ngIf="isVisible">This will be shown if isVisible is true.</div>
<ul>
  <li *ngFor="let item of items">{{ item }}</li>
</ul>

The *ngIf* directive conditionally includes a template, while the *ngFor* directive iterates over a list of items to display each one.

Services

Services in Angular are used to share data and functionality across different components. They are ideal for handling business logic, data access, and other reusable functionalities. Services are typically used in conjunction with Angular’s dependency injection system, which allows components and other services to be easily injected into each other.

Creating a Service

To create a new service, use the Angular CLI command ng generate service. This command generates a service with a TypeScript class that can be injected into components or other services. 


ng generate service my-service

The created service can be injected into components using Angular’s dependency injection system. For example: 


import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  private data: string[] = ['Item 1', 'Item 2', 'Item 3'];

  getData(): string[] {
    return this.data;
  }
}

In this example, the DataService class provides a method to retrieve an array of data. This service can be injected into components to access the data.

Summary

In this unit, you have explored the fundamental concepts of Angular, including components, directives, and services. These building blocks are essential for creating efficient and scalable Angular applications. Understanding how to use components to manage the user interface, directives to extend HTML functionality, and services to share data and logic will set a strong foundation for more advanced Angular topics.
