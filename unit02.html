Unit 02 Overview

Welcome to Unit 02 of DEV001: Angular Essential Training! In this unit, we will dive deeper into Angular components and their lifecycle. You will learn how to leverage Angular’s component lifecycle hooks to manage component behavior throughout its existence, from creation to destruction.

Understanding Component Architecture

Components in Angular are the fundamental building blocks of an application. Each component encapsulates its own HTML, CSS, and TypeScript code, making it a reusable and maintainable unit of code. Angular's component architecture is built on the following key principles:

  • Encapsulation: Components encapsulate their logic, view, and styles, which enhances modularity.
  • Reusability: Components can be reused across different parts of an application or even in different projects.
  • Composition: Components can be nested within other components to build complex UIs.

Component Lifecycle

Angular components go through a series of lifecycle phases from creation to destruction. Each phase offers lifecycle hooks that allow you to perform actions at specific points in the component’s lifecycle. Here’s a brief overview of the key lifecycle hooks:

  • ngOnInit: Called once after the component is initialized. Ideal for initialization logic.
  • ngOnChanges: Called when an input property value changes. Useful for reacting to changes in data-bound properties.
  • ngDoCheck: Called during every change detection run. Allows for custom change detection.
  • ngAfterContentInit: Called after Angular projects external content into the component’s view.
  • ngAfterContentChecked: Called after the content has been checked by the change detection mechanism.
  • ngAfterViewInit: Called after Angular initializes the component’s views and child views.
  • ngAfterViewChecked: Called after the view and child views have been checked.
  • ngOnDestroy: Called just before Angular destroys the component. Ideal for cleanup logic.

Example: Component Lifecycle Hooks

Here’s an example of an Angular component that utilizes several lifecycle hooks: 


import { Component, OnInit, OnDestroy, OnChanges, SimpleChanges } from '@angular/core';

@Component({
  selector: 'app-lifecycle-demo',
  template: '<div>Lifecycle Demo Component</div>',
})
export class LifecycleDemoComponent implements OnInit, OnDestroy, OnChanges {
  ngOnInit() {
    console.log('ngOnInit: Component initialized');
  }

  ngOnChanges(changes: SimpleChanges) {
    console.log('ngOnChanges: Changes detected', changes);
  }

  ngOnDestroy() {
    console.log('ngOnDestroy: Component about to be destroyed');
  }
}

In this example:

  • The ngOnInit hook logs a message when the component is initialized.
  • The ngOnChanges hook logs changes detected in input properties.
  • The ngOnDestroy hook logs a message when the component is about to be destroyed.

Best Practices for Using Lifecycle Hooks

To make the most of Angular’s lifecycle hooks, consider the following best practices:

  • Use ngOnInit for Initialization: Place initialization logic and data fetching in ngOnInit to ensure the component is fully set up before performing these actions.
  • Manage Resources in ngOnDestroy: Clean up resources, such as subscriptions and timers, in ngOnDestroy to avoid memory leaks.
  • Optimize Change Detection: Use ngDoCheck only if necessary, as it can impact performance due to frequent calls.
  • Avoid Expensive Operations in ngAfterViewInit: Avoid performing expensive operations in ngAfterViewInit, as it might affect the application’s performance.
  • Debounce Input Changes: Use ngOnChanges efficiently by debouncing frequent changes to avoid performance bottlenecks.
